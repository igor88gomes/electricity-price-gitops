1. Applikationen exponeras som ClusterIP på port {{ .Values.service.port }}.

2. Prova readiness:
   kubectl -n <ns> exec -it <pod> -- curl -s localhost:{{ .Values.service.targetPort }}{{ .Values.probes.readiness.path }}

3. Metrik (Prometheus):
   aktiverat via ServiceMonitor om serviceMonitor.enabled=true.

4. Ingress (om aktiverad):
   när .Values.ingress.enabled=true exponeras applikationen via Ingress.

   Exempel på hosts per miljö (kan variera beroende på values):
     - DEV:     http://dev.electricity-price.local
     - STAGING: http://staging.electricity-price.local
     - PROD:    http://prod.electricity-price.local

   I lokala kluster (t.ex. Minikube) kan du behöva lägga till motsvarande
   poster i /etc/hosts mot klustrets IP.
